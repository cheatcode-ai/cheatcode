# Mobile App Reference

## Mobile Interaction Patterns

- Bottom navigation for 3-5 primary destinations.
- Tabs for sibling views inside one workflow.
- Sheets for focused secondary actions.
- Segmented controls for mode switches.
- Steppers, sliders, and numeric inputs for numeric settings.
- Toasts for transient success; inline errors for form validation.

## QA Checklist

- No horizontal overflow at 390px width.
- Tap targets are not crowded.
- Keyboard does not cover the active input's submit action.
- Loading and empty states are mobile-sized.
- Primary action remains obvious after scrolling.
