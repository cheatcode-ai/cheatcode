# Landing Page Reference

## Required Sections

Use only the sections that serve the offer:

1. Hero with product or offer signal.
2. Problem and desired outcome.
3. Product workflow or feature proof.
4. Social proof, case study, or credibility signals.
5. Pricing or plan tease when relevant.
6. FAQ or objection handling.
7. Final CTA.

## Layout Rules

- Full-width page sections with constrained inner content.
- Cards only for repeated items, testimonials, pricing tiers, or framed tools.
- Hero text should not sit inside a card.
- Avoid nested cards.
- Use stable dimensions for repeated items.
- Make the next section visible from the first viewport.

## QA Checklist

- Desktop screenshot.
- Mobile screenshot.
- Console has no unexpected errors.
- Network has no failed critical assets.
- Longest nav item and CTA text fit.
- Hero, CTA, and proof are visible without instruction text.
