# Slide From PRD Reference

## Extraction Rules

- Preserve source terminology for core product concepts.
- Keep requirements separate from goals and open questions.
- Do not convert risks into solved problems.
- Map out-of-scope items to a slide note or appendix.
- Keep dates and milestones exact.

## Executive Slide Patterns

| Source section | Slide pattern |
|---|---|
| Background | One context slide with decision relevance |
| Goals | Success metrics and tradeoffs |
| Requirements | Capability clusters |
| User stories | Journey or workflow |
| Risks | Risk and mitigation matrix |
| Timeline | Milestone roadmap |

## Source-To-Slide Map

Create a short table:

| Source section | Slide | Notes |
|---|---|---|

This prevents accidental omission of important constraints.
